# Деплой на GitHub Pages

## Автоматический деплой

После push в репозиторий GitHub Pages автоматически задеплоит сайт, если включен в настройках.

## Настройка GitHub Pages

1. Перейдите в репозиторий: https://github.com/eveiljuice/sicilysite
2. Откройте **Settings** → **Pages**
3. В разделе **Source** выберите:
   - Branch: `main`
   - Folder: `/ (root)`
4. Нажмите **Save**

Через несколько минут сайт будет доступен по адресу:
**https://eveiljuice.github.io/sicilysite/**

## Проверка деплоя

После деплоя проверьте:
- ✅ Сайт открывается без ошибок
- ✅ Карта загружается (SVG)
- ✅ Анимации работают
- ✅ Модальные окна открываются
- ✅ Форма валидируется

## Обновление сайта

После каждого push в ветку `main` сайт автоматически обновится (обычно в течение 1-2 минут).

## Troubleshooting

### Сайт не обновляется
- Проверьте Actions в репозитории на наличие ошибок
- Убедитесь, что все пути относительные (начинаются с `./`)

### Карта не загружается
- Проверьте, что `assets/svg/sicily.svg` доступен
- Убедитесь, что CORS настроен правильно (GitHub Pages поддерживает CORS)

### Изображения не отображаются
- Добавьте реальные изображения в `assets/img/`
- Или используйте placeholder сервисы (picsum.photos, unsplash.com)
